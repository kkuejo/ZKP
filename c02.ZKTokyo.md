## 1. SNARKとは何か

**SNARK** は "Succinct Non-interactive ARgument of Knowledge" の略で、日本語にすると「簡潔な非対話型知識証明」です。

- **Succinct（簡潔）**：証明のサイズが小さい（回路の大きさに依らない）
- **Non-interactive（非対話型）**：証明者と検証者がリアルタイムでやり取りしなくてよい
- **ARgument of Knowledge（知識証明）**：秘密の情報（ウィットネス）を知っていることを証明できる

### 証明したいこと：回路充足性

SNARKが証明するのは「**算術回路**の充足性」です。

算術回路とは、加算（+）と乗算（×）のゲートで構成された計算グラフです。

$$C(x, w) = y$$

- $x$：公開入力（誰でも知っている値）
- $w$：秘密入力＝ウィットネス（証明者だけが知っている値）
- $y$：出力

**証明者の主張**：「$C(x, w) = y$ を満たす $w$ を知っている」

たとえば、ハッシュ関数の原像（プリイメージ）を知っていることの証明などに使われます。

## 2. SNARKの種類

SNARKはどの暗号技術を使うかによって分類されます。

| 暗号プリミティブ | セットアップ | 代表的スキーム |
|---|---|---|
| **ペアリング** | Trusted（信頼が必要） | Plonk, Groth16（Linear PCP） |
| **離散対数** | Transparent（信頼不要） | Bulletproofs, Halo2 |
| **ハッシュ** | Transparent（信頼不要） | STARK, Plonky3 |

### ペアリングとは？

ペアリングとは、楕円曲線上の2点を取り、別の群の1点に対応させる特殊な写像です。

$$e: \mathbb{G}_1 \times \mathbb{G}_2 \to \mathbb{G}_T$$

この写像が持つ「双線形性」を使うことで、暗号文のまま積の検証ができます。Groth16はこのペアリングを活用します。

### Trusted Setupとは？

Trusted Setup（信頼されたセットアップ）とは、証明システムの初期化時に「秘密の乱数 $\tau$」を使ってパラメータを生成し、その後 $\tau$ を完全に破棄するプロセスです。

$\tau$ が漏れると偽の証明を作れてしまうため、**誰も $\tau$ を知らない状態でセットアップを完了**することが重要です（マルチパーティ計算などで実現）。

## 3. Linear PCPベースSNARKの特徴

Groth16はLinear PCP（後述）をベースとしたSNARKです。その特徴を整理します。

### メリット

**✅ 証明サイズが小さい（100〜200バイト）**

回路の大きさがどれだけ大きくなっても、証明は3つの楕円曲線上の点（各32〜64バイト）で済みます。これはブロックチェーンへのオンチェーン検証に非常に有利です。

**✅ 検証が高速**

検証に必要な計算はペアリング演算数回だけです。

### デメリット

**❌ 証明生成に線形時間かかる（FFTが必要）**

証明生成はFFT（高速フーリエ変換）を含む多項式演算が必要で、回路サイズ $n$ に対して $O(n \log n)$ の計算量がかかります。

**❌ 回路ごとにTrusted Setupが必要**

Groth16は回路固有のセットアップパラメータを生成する必要があり、回路が変わるたびに新たなセットアップが必要です（Plonkはユニバーサルセットアップで改善）。

## 4. Linear PCPの歴史

| 年 | 論文/研究 | 内容 |
|---|---|---|
| 1992 | Kilian'92 | PCPとマークルツリーによるSNARKの原型 |
| 2000 | Micali'00 | Fiat-Shamir変換で非対話化 |
| 2007 | IKO'07 | **Linear PCP**と準同型暗号の組み合わせ（$O(n^2)$ の証明時間） |
| 2010 | Groth'10 | Linear PCPとペアリングの組み合わせ |
| 2012 | Lipmaa'12 | 証明パラメータ（gp）サイズの削減 |
| 2013 | GGPR'13 | **QSP/QAP**の導入（$O(n \log n)$ の証明時間に改善） |
| 2016 | **Groth16** | 最小の証明サイズ（3要素）と効率的な検証を実現 |

IKO'07でLinear PCPが誕生しましたが当初は証明時間が $O(n^2)$ と重く非実用的でした。GGPR'13でQAPを導入することで $O(n \log n)$ まで改善され、Groth16でさらに洗練されました。

## 5. Quadratic Arithmetic Program (QAP)

QAPは算術回路を「多項式の問題」に変換するための表現形式です。これがGroth16の核心的なアイデアです。

### 5-1. QAPの変換の流れ

$$\text{回路} \to \text{トレース} \to \text{セレクター多項式} \{l_i(x), r_i(x), o_i(x)\} \to \text{マスター多項式} \, p(x)$$

### 5-2. 例題の回路

講義では以下の回路を例として使います（入力 $w_1, \ldots, w_6$、乗算ゲート $g_1, g_4, g_5$、加算ゲート $g_2, g_3$）。

```
        g4        g5
       ×         ×
      / \       / \
    g1   g2   g2   g3
    ×    +    +    +
   /\   /\       /\
  w1 w2 w3 w4   w5 w6
```

具体的な関係：
- $g_1 = w_1 \times w_2$
- $g_2 = w_3 + w_4$
- $g_3 = w_5 + w_6$
- $g_4 = g_1 \times g_2$
- $g_5 = g_2 \times g_3$

### 5-3. トレース：何を記録するか

SNARKの証明ではまず「回路の計算の記録」（トレース）が必要です。証明方式によって何を記録するかが異なります。

| 方式 | 記録する内容 |
|---|---|
| Interactive Proof | 全てのゲートの値 |
| Plonk | 各ゲートの左入力・右入力・出力 |
| **QAP** | 入力値 $w_1, \ldots, w_6$ と乗算ゲートの出力 $g_1, g_4, g_5$ のみ |

**なぜQAPは加算ゲートの出力を記録しなくていいのか？**

加算ゲートの出力（例：$g_2 = w_3 + w_4$）は、入力が決まれば自動的に決まります。一方、乗算ゲートの出力は多項式の「制約」として別に扱う必要があります。後述のLinear PCPでは「ベクトルの線形結合」として評価するため、**非線形な乗算の結果だけを明示的に持てば十分**なのです。

QAPのトレースベクトル $\mathbf{c}$ は：

$$\mathbf{c} = (c_1, c_2, \ldots, c_9) = (w_1, w_2, w_3, w_4, w_5, w_6, g_1, g_4, g_5)$$

### 5-4. セレクター多項式

セレクター多項式は「ある変数 $c_i$ が、あるゲートの入力/出力であるかどうか」を多項式として表現したものです。

乗算ゲートが3つ（ゲート1, 2, 3に対応）あるので、評価点は $\omega, \omega^2, \omega^3$ の3点を使います（$\omega$ は有限体上の元）。

#### $l_i(x)$：$c_i$ がゲートの左入力かどうか

$$l_i(\omega^j) = \begin{cases} 1 & \text{$c_i$ がゲート$j$の左入力} \\ 0 & \text{それ以外} \end{cases}$$

例：$w_1$ はゲート1（$g_4 = g_1 \times g_2$ の $g_1$ 側）の左入力なので：

$$l_1(\omega) = 1, \quad l_1(\omega^2) = 0, \quad l_1(\omega^3) = 0$$

有限体 $\mathbb{F}_7$ で $\omega = 2$ とおくと、ラグランジュ補間により：

$$l_1(x) = 3x^2 + 6x + 5 \pmod{7}$$

> **ラグランジュ補間とは？** 複数の点 $(x_1, y_1), (x_2, y_2), \ldots$ を通る多項式を一意に求める手法です。$n$ 点を指定すると次数 $n-1$ 以下の多項式が一意に定まります。

#### $r_i(x)$：$c_i$ がゲートの右入力かどうか

同様に：

$$r_i(\omega^j) = \begin{cases} 1 & \text{$c_i$ がゲート$j$の右入力} \\ 0 & \text{それ以外} \end{cases}$$

#### $o_i(x)$：$c_i$ がゲートの出力かどうか

$$o_i(\omega^j) = \begin{cases} 1 & \text{$c_i$ がゲート$j$の出力} \\ 0 & \text{それ以外} \end{cases}$$

**加算ゲートの扱い（重要）**

加算ゲート $g_2 = w_3 + w_4$ が乗算ゲート $g_4$ の左入力になっている場合、$g_2$ 自体をトレースに含める代わりに、**$w_3$ と $w_4$ 両方をそのゲートの左入力として係数1で足す**ことで表現します。つまり $l_3(\omega^2) = 1, l_4(\omega^2) = 1$ となります。

有限体 $\mathbb{F}_7$ での全セレクター多項式（$\omega = 2$）：

| 変数 | $l_i(x)$ | $r_i(x)$ | $o_i(x)$ |
|---|---|---|---|
| $w_1$ | $3x^2+6x+5$ | $0$ | $0$ |
| $w_2$ | $0$ | $3x^2+6x+5$ | $0$ |
| $w_3$ | $5x^2+5x+5$ | $6x^2+3x+5$ | $0$ |
| $w_4$ | $5x^2+5x+5$ | $6x^2+3x+5$ | $0$ |
| $w_5$ | $0$ | $5x^2+5x+5$ | $0$ |
| $w_6$ | $0$ | $5x^2+5x+5$ | $0$ |
| $g_1$ | $6x^2+3x+5$ | $0$ | $3x^2+6x+5$ |
| $g_4$ | $0$ | $0$ | $6x^2+3x+5$ |
| $g_5$ | $0$ | $0$ | $5x^2+5x+5$ |

### 5-5. マスター多項式 $p(x)$

トレースベクトル $\mathbf{c} = (c_1, \ldots, c_m)$ とセレクター多項式を使って、以下の「マスター多項式」を定義します：

$$p(x) = L(x) \cdot R(x) - O(x)$$

ただし：

$$L(x) = \sum_{i=1}^{m} c_i \cdot l_i(x), \quad R(x) = \sum_{i=1}^{m} c_i \cdot r_i(x), \quad O(x) = \sum_{i=1}^{m} c_i \cdot o_i(x)$$

**$p(x)$ の意味**

各乗算ゲート $j$ について、評価点 $\omega^j$ でのマスター多項式の値を見ると：

$$p(\omega^j) = L(\omega^j) \cdot R(\omega^j) - O(\omega^j)$$
$$= (\text{ゲート$j$の左入力の和}) \times (\text{ゲート$j$の右入力の和}) - (\text{ゲート$j$の出力})$$

回路が正しく計算されている（充足されている）場合、各乗算ゲートで「左入力 × 右入力 = 出力」が成り立つため：

$$p(\omega^j) = 0 \quad \text{for all } j = 1, 2, 3$$

**証明者の主張**：$p(\omega) = 0, \, p(\omega^2) = 0, \, p(\omega^3) = 0$

### 5-6. 消失多項式 $V(x)$

$p(x)$ が $\omega, \omega^2, \omega^3$ の全てで0になるということは、$p(x)$ はこれらの点を根として持つ多項式で割り切れることを意味します。

**消失多項式**（Vanishing Polynomial）を：

$$V(x) = (x - \omega)(x - \omega^2)(x - \omega^3)$$

と定義すると、$p(\omega^j) = 0$ iff $V(x) \mid p(x)$ なので：

$$p(x) = V(x) \cdot q(x)$$

となる多項式 $q(x)$ が存在します。

### 5-7. 回路充足性とQAPの等価性

$$\underbrace{C(x, w) = y \text{ を満たす } w \text{ を知っている}}_{\text{回路充足性}} \iff \underbrace{p(x) = V(x)q(x) \text{ となる } q(x) \text{ を知っている}}_{\text{QAP}}$$

これで問題が「多項式が特定の多項式で割り切れるか」という形に変換されました。

## 6. Linear PCP

### 6-1. PCP（Probabilistically Checkable Proof）とは

通常の証明は全部を確認する必要がありますが、PCPオラクルは「ランダムに選んだ一部のポイントだけを確認することで、全体の正しさを確率的に検証できる」仕組みです。

証明（巨大な配列）をオラクル（クラウド）に預け、検証者がランダムなインデックスを指定して値を確認します。仮に証明が偽物だとしても、高確率で矛盾が検出されます。

### 6-2. Linear PCP オラクル

Linear PCP では、オラクルが保持するのは**ベクトル $\mathbf{c}$**（トレース）で、検証者からのクエリは「あるベクトル $\mathbf{q}$ との内積を教えて」という形です：

$$\langle \mathbf{c}, \mathbf{q} \rangle = \sum_i c_i \cdot q_i$$

これが「線形（Linear）」と呼ばれる理由です。

### 6-3. QAPとLinear PCPの組み合わせ

マスター多項式の等式 $p(x) = V(x)q(x)$ を、ランダム点 $\gamma$ で評価すると：

$$L(\gamma) \cdot R(\gamma) - O(\gamma) = V(\gamma) \cdot q(\gamma)$$

各項を内積で表すと：
- $L(\gamma) = \langle \mathbf{c}, (l_1(\gamma), l_2(\gamma), \ldots, l_m(\gamma)) \rangle = \langle \mathbf{c}, \mathbf{l}(\gamma) \rangle$
- $R(\gamma) = \langle \mathbf{c}, \mathbf{r}(\gamma) \rangle$
- $O(\gamma) = \langle \mathbf{c}, \mathbf{o}(\gamma) \rangle$
- $V(\gamma) \cdot q(\gamma) = V(\gamma) \cdot \langle \mathbf{q}, \boldsymbol{\gamma} \rangle$（$\mathbf{q}$ は $q(x)$ の係数ベクトル）

検証者は：

$$\langle \mathbf{c}, \mathbf{l}(\gamma) \rangle \times \langle \mathbf{c}, \mathbf{r}(\gamma) \rangle - \langle \mathbf{c}, \mathbf{o}(\gamma) \rangle = V(\gamma) \langle \mathbf{q}, \boldsymbol{\gamma} \rangle$$

が成立するかを確認します。

## 7. Linear PCPによるSNARK

Linear PCPをSNARKに変換するために、**ペアリングベースの暗号コミットメント**を使います。

### 7-1. セットアップフェーズ

$\gamma = \tau$（秘密の乱数）として、以下のパラメータを生成します。

**証明鍵（PK）**：

$$\text{PK} = \left\{ p, \mathbb{G}, g, \mathbb{G}_T, e, \; g^{l_i(\tau)}, g^{r_i(\tau)}, g^{o_i(\tau)}, g^{\tau^i} \; (\forall i = 1, \ldots, m) \right\}$$

- $g$：楕円曲線の生成元
- $g^{l_i(\tau)}$：$l_i(\tau)$ を指数に持つ群の元（離散対数問題により $l_i(\tau)$ を逆算できない）
- $g^{\tau^i}$：$\tau$ のべき乗を指数に持つ元（多項式評価に使う）

**検証鍵（VK）**：

$$\text{VK} = \left\{ g, \; g^{V(\tau)} \right\}$$

$\tau$ はこの後完全に破棄されます（Trusted Setup）。

### 7-2. 証明生成フェーズ

証明者はPKを使って以下の4つのコミットメントを計算します：

$$\pi_1 = g^{L(\tau)} = g^{\sum_i c_i \cdot l_i(\tau)} = \prod_i \left(g^{l_i(\tau)}\right)^{c_i}$$

$$\pi_2 = g^{R(\tau)} = \prod_i \left(g^{r_i(\tau)}\right)^{c_i}$$

$$\pi_3 = g^{O(\tau)} = \prod_i \left(g^{o_i(\tau)}\right)^{c_i}$$

$$\pi_4 = g^{q(\tau)} = \prod_i \left(g^{\tau^i}\right)^{q_i}$$

証明 $\pi = (\pi_1, \pi_2, \pi_3, \pi_4)$

> **ポイント**：証明者は $\tau$ を知らなくても、$g^{l_i(\tau)}$ や $g^{\tau^i}$ があれば、自分の秘密 $c_i$ を使って $g^{L(\tau)}$ を計算できます。

### 7-3. 検証フェーズ

検証者はペアリング関数 $e: \mathbb{G} \times \mathbb{G} \to \mathbb{G}_T$ を使って：

$$\frac{e(\pi_1, \pi_2)}{e(\pi_3, g)} \stackrel{?}{=} e\left(g^{V(\tau)}, \pi_4\right)$$

これを展開すると：

$$e\left(g^{L(\tau)}, g^{R(\tau)}\right) / e\left(g^{O(\tau)}, g\right) = e\left(g^{V(\tau)}, g^{q(\tau)}\right)$$

ペアリングの双線形性 $e(g^a, g^b) = e(g, g)^{ab}$ を使うと：

$$e(g,g)^{L(\tau) \cdot R(\tau) - O(\tau)} = e(g,g)^{V(\tau) \cdot q(\tau)}$$

つまり $p(\tau) = V(\tau) q(\tau)$ を検証していることになります。

### 7-4. PKの正当利用を保証する問題

ここで問題が生じます。証明者が PK の $g^{l_i(\tau)}$ などを使って正直に $\pi_1, \pi_2, \pi_3$ を計算していることをどう確認するか？

不正な証明者は、例えば $\pi_1$ の計算に $g^{l_i(\tau)}$ を使わず、全く無関係な値を $\pi_3$ に入れて辻褄を合わせようとするかもしれません。

**解決策**：PKに追加のパラメータ $g^\beta, g^{\beta(l_i(\tau) + r_i(\tau) + o_i(\tau))}$ を加え、証明者は：

$$\pi_5 = \prod_i \left(g^{\beta(l_i(\tau) + r_i(\tau) + o_i(\tau))}\right)^{c_i}$$

を計算します。検証者は：

$$e(\pi_1 \pi_2 \pi_3, g^\beta) = e(\pi_5, g)$$

を追加チェックすることで、$\pi_1, \pi_2, \pi_3$ が同じ $c_i$ から計算されていることを保証します。

## 8. Groth16

### 8-1. R1CS（Rank-1 Constraint System）

Groth16はQAPと等価な「R1CS」という表現を使います。

QAPはゲートごとに記述しますが、R1CSは**制約ごとに行列形式で記述**します。

QAPの制約：
$$w_1 \times w_2 - g_1 = 0$$

R1CSでは線形結合のより一般的な形：
$$(3w_1 + 5w_5 - 7g_1) \times (6w_2 + 10g_5) - (w_3 - 2g_4) = 0$$

行列形式では：
$$(\mathbf{L} \cdot \mathbf{c}) \otimes (\mathbf{R} \cdot \mathbf{c}) = \mathbf{O} \cdot \mathbf{c}$$

ここで $\mathbf{L}, \mathbf{R}, \mathbf{O}$ は $n \times m$ 行列（$n$：制約数、$m$：変数数）。

### 8-2. Groth16の構成

Groth16は前節のLinear PCPベースSNARKを改良し、**証明要素を3つに削減**しました。

**新たな秘密パラメータ**：$\alpha, \beta$（セットアップ時に追加）

**証明鍵（PK）**：

$$\text{PK} = \left\{ p, \mathbb{G}, g, \mathbb{G}_T, e, \; g^{l_i(\tau)}, g^{r_i(\tau)}, g^{o_i(\tau)}, g^{\tau^i}, g^\alpha, g^\beta, g^{\beta l_i(\tau) + \alpha r_i(\tau) + o_i(\tau)} \right\}$$

**検証鍵（VK）**：

$$\text{VK} = \left\{ g, \; g^\alpha, \; g^\beta \right\}$$

**証明**：

$$\pi_1 = g^{\alpha + \sum_i c_i \cdot l_i(\tau)}$$

$$\pi_2 = g^{\beta + \sum_i c_i \cdot r_i(\tau)}$$

$$\pi_3 = g^{\sum_i c_i \cdot (\beta l_i(\tau) + \alpha r_i(\tau) + o_i(\tau)) + V(\tau) q(\tau)}$$

**検証**：

$$\frac{e(\pi_1, \pi_2)}{e(\pi_3, g)} \stackrel{?}{=} e\left(g^\alpha, g^\beta\right)$$

展開すると：

$$e(g,g)^{(\alpha + L(\tau))(\beta + R(\tau)) - (C(\tau) + V(\tau)q(\tau))} = e(g,g)^{\alpha\beta}$$

ここで $C(\tau) = \sum_i c_i(\beta l_i(\tau) + \alpha r_i(\tau) + o_i(\tau))$ とおくと：

$$\alpha\beta + \alpha R(\tau) + \beta L(\tau) + L(\tau)R(\tau) - C(\tau) - V(\tau)q(\tau) = \alpha\beta$$

$$\alpha R(\tau) + \beta L(\tau) + L(\tau)R(\tau) - (\beta L(\tau) + \alpha R(\tau) + O(\tau)) - V(\tau)q(\tau) = 0$$

$$L(\tau)R(\tau) - O(\tau) = V(\tau)q(\tau)$$

これはまさに $p(\tau) = V(\tau)q(\tau)$ の検証です！$\alpha, \beta$ はキャンセルされ、結局QAPの検証と等価になります。

### 8-3. ゼロ知識性の達成

上記の証明はQAPが満たされていることは証明できますが、証明から秘密 $\mathbf{c}$ が漏れる可能性があります。

ゼロ知識性を達成するには、$V(x)$ で割り切れるランダム値 $\delta_1, \delta_2, \delta_3$ を足します：

| 元の証明 | ゼロ知識版 |
|---|---|
| $\pi_1 = g^{\sum c_i \cdot l_i(\tau)}$ | $\pi_1 = g^{\sum c_i \cdot l_i(\tau) + \delta_1 V(\tau)}$ |
| $\pi_2 = g^{\sum c_i \cdot r_i(\tau)}$ | $\pi_2 = g^{\sum c_i \cdot r_i(\tau) + \delta_2 V(\tau)}$ |
| $\pi_3 = g^{\sum c_i \cdot o_i(\tau)}$ | $\pi_3 = g^{\sum c_i \cdot o_i(\tau) + \delta_3 V(\tau)}$ |
| $\pi_4 = g^{q(\tau)}$ | $\pi_4 = g^{q(\tau)}$（変わらず） |

**なぜこれでゼロ知識になるか？**

$V(x)$ で割り切れる項を足しても $p(x) = V(x)q(x)$ の成立には影響しません（$q(x)$ が少し変わるだけ）。一方、$\delta_1, \delta_2, \delta_3$ はランダムなので、証明を見ても秘密 $\mathbf{c}$ を逆算できなくなります。

### 8-4. Groth16の完全な仕様

最終的なGroth16の構成：

| | 内容 |
|---|---|
| **証明サイズ** | $(\pi_1, \pi_2, \pi_3)$：3つの群元（約192バイト） |
| **検証** | ペアリング1回の比較 $e(\pi_1, \pi_2) / e(\pi_3, g) = e(g^\alpha, g^\beta)$ |
| **セキュリティ仮定** | Generic Group Model + KoE（Knowledge of Exponent）仮定 |

## 9. まとめ

### Groth16の全体像

$$\text{算術回路 } C \xrightarrow{\text{QAP変換}} \text{多項式問題} \xrightarrow{\text{Linear PCP}} \text{群上のコミットメント} \xrightarrow{\text{ペアリング検証}} \text{SNARK}$$

**3つのキーアイデア**：

1. **QAP**：回路充足性を「$p(x) = V(x)q(x)$ となる $q$ を知っている」という多項式問題に変換
2. **Linear PCP**：秘密のトレース $\mathbf{c}$ を楕円曲線上にコミットし、$\tau$ での評価を暗号的に検証
3. **ペアリング**：積の検証を暗号文のまま行う

**Groth16の強み**：
- 証明サイズ：わずか3つの楕円曲線点（約192バイト）
- 検証：ペアリング演算1回
- EthereumのEIP-197でネイティブサポートされており、Circomで広く使われている

**Groth16の弱点**：
- 回路ごとにTrusted Setupが必要
- 証明生成：$O(n \log n)$（FFT）

## 10. 演習問題と解答

### 問1：QAPのトレースがなぜ加算ゲートの出力を記録しなくてもいいか

**解答**：

Linear PCPでは、証明者は検証者のクエリに対して「秘密ベクトル $\mathbf{c}$ との線形結合（内積）」を返します。

加算ゲートの出力 $g_2 = w_3 + w_4$ は、$w_3$ と $w_4$ の**線形結合**です。したがって、$w_3$ と $w_4$ が既にトレースに含まれていれば、$g_2$ の情報はセレクター多項式を使って$L(x), R(x), O(x)$の中に線形結合として吸収できます。

具体的には、$g_2$ が乗算ゲートの左入力となる場合、$l_{w_3}(\omega^j) = 1$ かつ $l_{w_4}(\omega^j) = 1$ とすることで $L(\omega^j) = c_{w_3} + c_{w_4} = g_2$ を実現できます。

一方、乗算ゲートの出力は $L(\omega^j) \times R(\omega^j) = O(\omega^j)$ という**非線形な制約**を満たす必要があるため、明示的にトレースに記録する必要があります。

### 問2：マスター多項式 $p(x)$ と商多項式 $q(x)$ の例

**解答**（簡単な例）：

回路：$g_1 = w_1 \times w_2$（乗算ゲート1つのみ）として $w_1 = 3, w_2 = 4, g_1 = 12$。

トレース：$\mathbf{c} = (w_1, w_2, g_1) = (3, 4, 12)$

セレクター多項式（1ゲートなので $\omega$ の1点のみ）：
- $l_1(x) = 1$（$w_1$ はゲート1の左入力）
- $r_2(x) = 1$（$w_2$ はゲート1の右入力）
- $o_3(x) = 1$（$g_1$ はゲート1の出力）

すると：

$$L(x) = 3, \quad R(x) = 4, \quad O(x) = 12$$

$$p(x) = L(x) \cdot R(x) - O(x) = 3 \cdot 4 - 12 = 0$$

消失多項式 $V(x) = (x - \omega)$。

$$q(x) = p(x) / V(x) = 0 / (x - \omega) = 0$$

$p(x) = 0 = V(x) \cdot 0$、つまり $q(x) = 0$ となり、回路の充足性が証明されます。

### 問3：安全性仮定

**解答**：

**Generic Group Model (GGM)**

GGMは「群の元に対してラベルのみを見えるようにし、演算結果は神託（オラクル）に問い合わせる」というモデルです。このモデルでは、群の特定の代数的構造を利用した攻撃が不可能と仮定し、Groth16の健全性（Soundness）と知識健全性（Knowledge Soundness）が証明されています。

**Knowledge of Exponent (KoE) 仮定**

「$g^a$ と $g^b$ が与えられたとき、$g^{ab}$ を出力できる者は $b$ を知っている」という仮定です。これにより、証明者が $\pi_1, \pi_2, \pi_3$ を正しく PK の要素から計算していることが保証されます。

これらの仮定の下で、Groth16は以下を満たすことが証明されています：
- **完全性（Completeness）**：正直な証明者は常に受理される
- **健全性（Soundness）**：偽の証明者は（圧倒的確率で）受理されない
- **ゼロ知識性（Zero-Knowledge）**：検証者は $\mathbf{c}$ に関する情報を何も得られない
